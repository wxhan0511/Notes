# 在嵌入式项目开发中，常见的目录结构及各模块代码归类建议如下：

- bsp（Board Support Package，板级支持包）：放与具体硬件板相关的初始化、配置代码，如时钟、GPIO、外设驱动（bsp.c、bsp_xxx.c）。
- core：放内核相关代码，如启动文件、系统初始化、RTOS移植层等（system_xxx.c、startup_xxx.s）。
- user（或 app）：放用户应用逻辑代码，如主循环、业务流程、界面处理（main.c、app_xxx.c）。
- driver：放通用外设驱动代码，如 SPI、I2C、UART、ADC、定时器等（driver_xxx.c）。
- utils（或 common）：放工具类、通用算法、数据结构、辅助函数（utils.c、crc.c、ring_buffer.c）。
- inc（include）：放所有头文件，按模块分类（bsp/inc、driver/inc、user/inc）。
- src：放所有源文件，按模块分类（bsp/src、driver/src、user/src）。
- config：放配置文件，如硬件参数、宏定义、外设映射（config.h）。
- Libraries：放第三方库或芯片厂商提供的库（如 STM32 HAL、CMSIS）。

举例：
```
Project/
  ├── bsp/
  │     ├── inc/
  │     └── src/
  ├── core/
  ├── user/
  │     ├── inc/
  │     └── src/
  ├── driver/
  │     ├── inc/
  │     └── src/
  ├── utils/
  ├── config/
  ├── Libraries/
  └── main.c
```

这样分层有利于代码复用、维护和团队协作。实际项目可根据规模和需求灵活调整。



# 对于 RTOS（如 FreeRTOS、RTX、ThreadX）嵌入式工程，推荐的详细目录结构及各模块归类如下：

1. bsp（Board Support Package）
板级初始化、硬件抽象层、外设底层驱动（如时钟、GPIO、串口、SPI、I2C、ADC、定时器等）。
例：bsp/inc/bsp_gpio.h、bsp/src/bsp_gpio.c
2. core
芯片启动文件、系统初始化、RTOS移植层（如 startup_xxx.s、system_xxx.c）。
RTOS相关的移植代码（如 FreeRTOSConfig.h、port.c）。
3. rtos
RTOS内核源码（可用子目录存放官方源码或移植层）。
RTOS任务创建、调度、信号量、消息队列等相关代码。
例：rtos/tasks.c、rtos/rtos_init.c
4. user 或 app
用户应用层代码，主要业务逻辑、主任务入口、各功能模块的任务实现。
例：user/inc/app_main.h、user/src/app_main.c、user/src/task_display.c
5. driver
通用外设驱动（与芯片无关的抽象驱动），如传感器、屏幕、存储器等。
例：driver/inc/ssd1306.h、driver/src/ssd1306.c
6. utils 或 common
工具类、通用算法、数据结构、调试辅助、日志、CRC、环形缓冲区等。
例：utils/log.c、utils/ring_buffer.c
7. config
配置文件、宏定义、硬件参数、外设映射、RTOS参数等。
例：config/pinmap.h、config/board_config.h
8. include/inc
头文件统一管理，按模块分类。
9. src
源文件统一管理，按模块分类。
10. Libraries
芯片厂商库、第三方库（如 STM32 HAL、CMSIS、FatFS、USB、LWIP）。
11. tasks 或 thread
各 RTOS 任务（线程）实现，建议每个主要功能一个任务文件。
例：tasks/task_comm.c、tasks/task_sensor.c
12. middleware
协议栈、中间件（如 TCP/IP、USB、文件系统、GUI）。
13. doc
项目文档、说明书、协议文档。
14. test 或 examples
测试代码、示例代码。
RTOS 工程推荐目录结构示例

Project/
├── bsp/
│   ├── inc/
│   └── src/
├── core/
├── rtos/
│   ├── kernel/
│   ├── port/
│   └── config/
├── user/
│   ├── inc/
│   └── src/
├── driver/
│   ├── inc/
│   └── src/
├── tasks/
│   ├── task_comm.c
│   ├── task_sensor.c
│   └── ...
├── utils/
├── config/
├── Libraries/
├── middleware/
├── doc/
├── test/
└── main.c
